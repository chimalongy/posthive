⚠TikTok is not like any other platform. Facebook, Instagram, and Twitter accept a single POST with a media URL. TikTok requires a mandatory 4-step pipeline: creator info query → init → video transfer → async status polling. You cannot skip or reorder any step. Silent failures are common if you do.
STEP 0Token refresh — run before every single upload
0
Refresh the access token before starting the pipeline 24hr TTL
Access tokens expire every 24 hours. Check expiry and refresh proactively — never wait for a 401 error mid-upload.
⌄
  • When credentials were stored during OAuth, you should have saved token_expires_at = Date.now() + (expires_in * 1000).
  • Before starting any upload pipeline, compare Date.now() with token_expires_at. If within 5 minutes of expiry — refresh immediately.
  • Do NOT wait for the API to return a 401. By then your pipeline is already broken mid-upload.
  • Both tokens rotate on every refresh. The old access_token and refresh_token become invalid immediately. Save the new pair back to your database before continuing.
// lib/platformPosters/tiktok.js
async function refreshIfNeeded(credentials) {
  const BUFFER = 5 * 60 * 1000; // 5 minutes buffer
  if (Date.now() < credentials.token_expires_at - BUFFER) {
    return credentials; // still valid, continue
  }

  const res = await fetch('https://open.tiktokapis.com/v2/oauth/token/', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      client_key:    credentials.client_key,
      client_secret: credentials.client_secret,
      grant_type:    'refresh_token',
      refresh_token: credentials.refresh_token,
    }),
  });

  const d = await res.json();
  if (!d.access_token) throw new Error('TikTok token refresh failed: ' + JSON.stringify(d));

  const updated = {
    ...credentials,
    access_token:     d.access_token,
    refresh_token:    d.refresh_token,   // new refresh token — old one is now invalid
    token_expires_at: Date.now() + (d.expires_in * 1000),
  };

  // Save updated tokens back to Supabase immediately
  // await supabase.from('connected_platforms')
  //   .update({ credentials: updated }).eq('id', platformRecordId);

  return updated;
}
ℹIf the refresh_token itself has expired (365-day TTL), there is no silent recovery. The user must re-authorize your app through the full OAuth flow again. Build a re-auth prompt into your UI for this case.
STEP 1Query creator info — mandatory before every upload init
1
POST /v2/post/publish/creator_info/query/ Required by TikTok
Returns the creator's allowed privacy levels, max video duration, interaction capabilities. Must be called before Step 2 — not all creators have the same posting options.
⌄
  • This call is mandatory before every upload. It returns what this specific creator account is allowed to post — privacy options vary per account.
  • Use privacy_level_options[0] as your default privacy when building the init request in Step 2.
  • In sandbox, this call will return only SELF_ONLY in privacy_level_options — this is expected. It means posts go to the creator's private inbox.
  • Rate limit: 6 requests per minute per user access_token.
POST https://open.tiktokapis.com/v2/post/publish/creator_info/query/
Authorization: Bearer act.YOUR_ACCESS_TOKEN
Content-Type: application/json; charset=UTF-8

// No request body needed
// Response
{
  "data": {
    "creator_username":           "@youruser",
    "privacy_level_options":       ["PUBLIC_TO_EVERYONE", "MUTUAL_FOLLOW_FRIENDS", "SELF_ONLY"],
    "duet_disabled":              false,
    "stitch_disabled":            false,
    "comment_disabled":           false,
    "max_video_post_duration_sec": 600   // 10 minutes max for this creator
  },
  "error": { "code": "ok" }
}
async function queryCreatorInfo(accessToken) {
  const res = await fetch('https://open.tiktokapis.com/v2/post/publish/creator_info/query/', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${accessToken}`,
      'Content-Type':  'application/json; charset=UTF-8',
    },
  });
  const d = await res.json();
  if (d.error?.code !== 'ok') throw new Error('Creator info failed: ' + d.error?.message);
  return d.data;
}
STEP 2Initialize upload — choose PULL_FROM_URL or FILE_UPLOAD
2
Initialize the upload — returns publish_id and upload_url upload_url valid 1 hour only
Two source methods. PULL_FROM_URL is recommended for PostHive — video is already in Supabase Storage with a public URL. FILE_UPLOAD requires chunked PUT requests in Step 3.
⌄

Use the right endpoint based on sandbox vs production:

ModeEndpointResult
Sandbox/v2/post/publish/inbox/video/init/Sent to creator's private TikTok inbox as a draft. Never public.
Production/v2/post/publish/video/init/Posted to creator's TikTok profile (per privacy_level setting).

Method A — PULL_FROM_URL (recommended for PostHive)

  • TikTok's servers fetch the video directly from your public Supabase Storage URL. You only send the URL — no file bytes transferred from your server.
  • No chunking logic required. No Content-Range headers. Far simpler implementation.
  • The URL must be https://, publicly accessible without auth headers, and the domain must be verified in TikTok Developer Portal URL properties. Add your Supabase project storage domain (e.g. xxxx.supabase.co) to URL properties.
  • The URL must not redirect to another URL.
// Method A — PULL_FROM_URL (Production)
POST https://open.tiktokapis.com/v2/post/publish/video/init/
Authorization: Bearer act.YOUR_ACCESS_TOKEN
Content-Type: application/json; charset=UTF-8

{
  "post_info": {
    "title":                   "Your caption #hashtag",
    "privacy_level":           "PUBLIC_TO_EVERYONE",  // use privacy_level_options[0] from Step 1
    "disable_duet":            false,
    "disable_comment":         false,
    "disable_stitch":          false,
    "video_cover_timestamp_ms": 1000   // thumbnail frame at 1 second
  },
  "source_info": {
    "source":    "PULL_FROM_URL",
    "video_url": "https://xxxx.supabase.co/storage/v1/object/public/media-uploads/..."
  }
}

Method B — FILE_UPLOAD (when sending raw video bytes from your server)

  • Use when the video is not yet on a public CDN. You calculate chunk math and send binary data in Step 3.
  • Must provide video_size (total bytes), chunk_size, and total_chunk_count before calling init.
// Method B — FILE_UPLOAD (requires chunked PUTs in Step 3)
{
  "post_info": { /* same as above */ },
  "source_info": {
    "source":            "FILE_UPLOAD",
    "video_size":        50000123,   // total file bytes
    "chunk_size":        10000000,   // bytes per chunk — min 5MB, max 64MB
    "total_chunk_count": 5           // = Math.ceil(video_size / chunk_size)
  }
}

Init response — same for both methods:

{
  "data": {
    "publish_id":  "v_pub_file~v2-1.123456789",  // ← save this for Step 4 status polling
    "upload_url":  "https://open-upload.tiktokapis.com/video/?upload_id=67890&upload_token=Xza123"
                                                    // ← FILE_UPLOAD only. Valid for 1 hour.
  },
  "error": { "code": "ok", "message": "" }
}
⚠The upload_url is valid for 1 hour only. Complete all chunk uploads within that window. If it expires, call /init/ again for a new URL and a new publish_id.
STEP 3Transfer video — chunked PUT or skipped (URL pull)
3
Transfer video bytes to TikTok's upload server
PULL_FROM_URL users: TikTok fetches your Supabase URL automatically — skip to Step 4. FILE_UPLOAD users: send each chunk with a sequential PUT request.
⌄
✓PULL_FROM_URL — skip this step entirely. TikTok fetches the file from your Supabase Storage URL. Go straight to Step 4 polling.

FILE_UPLOAD — chunk rules you must follow exactly:

RuleValue / Requirement
Min chunk size5 MB per chunk (except final chunk)
Max chunk size64 MB per chunk
Final chunkCan be up to 128 MB to absorb trailing bytes below 5 MB
Files under 5 MBUpload as one whole chunk — set chunk_size = video_size, total_chunk_count = 1
Files over 64 MBMust be split into multiple chunks
Max chunksMinimum 1, maximum 1000
Upload orderSequential only. Await each PUT before starting the next. Parallel uploads are NOT supported.
Same upload URLUse the exact same upload_url from Step 2 for every chunk, including all query parameters.
Sandbox file limit128 MB max in sandbox. Production allows up to 4 GB.

Chunk calculation example — 50,000,123 byte file with 10 MB chunks:

video_size        = 50000123
chunk_size        = 10000000   // 10 MB
total_chunk_count = 5          // Math.ceil(50000123 / 10000000)

// Content-Range header for each chunk:
Chunk 1: bytes 0–9999999/50000123
Chunk 2: bytes 10000000–19999999/50000123
Chunk 3: bytes 20000000–29999999/50000123
Chunk 4: bytes 30000000–39999999/50000123
Chunk 5: bytes 40000000–50000122/50000123  // final: includes trailing 123 bytes

PUT request format for each chunk:

PUT https://open-upload.tiktokapis.com/video/?upload_id=67890&upload_token=Xza123
Content-Type: video/mp4
Content-Length: {BYTE_SIZE_OF_THIS_CHUNK}
Content-Range: bytes {FIRST_BYTE}-{LAST_BYTE}/{TOTAL_BYTE_LENGTH}

[BINARY DATA OF THIS CHUNK]

Node.js chunk upload implementation:

async function uploadInChunks(uploadUrl, fileBuffer) {
  const CHUNK_SIZE  = 10 * 1024 * 1024; // 10 MB
  const totalSize   = fileBuffer.byteLength;
  const totalChunks = Math.ceil(totalSize / CHUNK_SIZE);

  for (let i = 0; i < totalChunks; i++) {
    const start = i * CHUNK_SIZE;
    const end   = Math.min(start + CHUNK_SIZE, totalSize);
    const chunk = fileBuffer.slice(start, end);

    const res = await fetch(uploadUrl, {
      method: 'PUT',
      headers: {
        'Content-Type':   'video/mp4',
        'Content-Length': String(chunk.byteLength),
        'Content-Range':  `bytes ${start}-${end - 1}/${totalSize}`,
      },
      body: chunk,
    });

    if (!res.ok) {
      throw new Error(`Chunk ${i + 1}/${totalChunks} upload failed: HTTP ${res.status}`);
    }
    // Check response header Content-Range to confirm received bytes
  }
}

STEP 4Poll publish status — async wait for completion
4
Poll /v2/post/publish/status/fetch/ using publish_id Async — use exponential backoff
TikTok processes and moderates content asynchronously. You are not given the post_id until complete. Poll with backoff — 30 requests/minute limit.
⌄
  • After Step 2 (PULL_FROM_URL) or Step 3 (FILE_UPLOAD), TikTok begins downloading and processing the video asynchronously. There is no synchronous publish confirmation.
  • Use the publish_id from Step 2 to poll the status endpoint. Rate limit: 30 requests per minute per access_token.
  • Use exponential backoff. Start at 3 second intervals, increase by 1.5× each poll, cap at 30 seconds. Polling every second will hit rate limits fast.
  • For public posts, moderation usually completes within 1 minute but can take several hours in edge cases.

All possible status values:

PROCESSING_DOWNLOAD
TikTok is downloading from your Supabase URL (PULL_FROM_URL). Normal — keep polling.
PROCESSING_UPLOAD
File chunks received, TikTok is processing. Normal — keep polling.
AWAIT_POSTING
Upload done, awaiting content moderation. Normal for public posts — keep polling.
SEND_TO_USER_INBOX
Sent to creator's inbox as a draft. Final success state in sandbox. Never progresses further in sandbox.
PUBLISH_COMPLETE
🎉 Video is live on TikTok. publicaly_available_post_id is returned. Final success state in production.
FAILED
Failed during processing or moderation. Check fail_reason. Do not retry same content automatically.
POST https://open.tiktokapis.com/v2/post/publish/status/fetch/
Authorization: Bearer act.YOUR_ACCESS_TOKEN
Content-Type: application/json; charset=UTF-8

{ "publish_id": "v_pub_file~v2-1.123456789" }
// Success response (production)
{
  "data": {
    "status":                       "PUBLISH_COMPLETE",
    "publicaly_available_post_id":   ["7123456789012345678"],  // note: TikTok typo in field name
    "uploaded_bytes":               50000123
  },
  "error": { "code": "ok" }
}

// Failure response
{
  "data": {
    "status":      "FAILED",
    "fail_reason": "video_duration_check_failed"
  }
}
async function pollStatus(accessToken, publishId, isSandbox, maxAttempts = 20) {
  let delay = 3000;

  for (let attempt = 0; attempt < maxAttempts; attempt++) {
    await new Promise(r => setTimeout(r, delay));

    const res = await fetch('https://open.tiktokapis.com/v2/post/publish/status/fetch/', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type':  'application/json; charset=UTF-8',
      },
      body: JSON.stringify({ publish_id: publishId }),
    });

    const d    = await res.json();
    const status = d.data?.status;

    if (status === 'PUBLISH_COMPLETE')
      return { success: true,  postId: d.data.publicaly_available_post_id?.[0] };

    if (status === 'SEND_TO_USER_INBOX' && isSandbox)
      return { success: true,  postId: null, note: 'Sandbox: sent to TikTok inbox as draft' };

    if (status === 'FAILED')
      return { success: false, error: d.data?.fail_reason || 'Processing failed' };

    // Still processing — back off
    delay = Math.min(delay * 1.5, 30000);
  }

  return { success: false, error: 'Polling timed out after max attempts' };
}
🧪Critical sandbox note: PUBLISH_COMPLETE is never reached in sandbox — public posting is disabled. Treat SEND_TO_USER_INBOX as your sandbox success state. If your polling logic only checks for PUBLISH_COMPLETE, it will time out on every sandbox test.

// lib/platformPosters/tiktok.js
// PostHive — Complete TikTok poster (PULL_FROM_URL, sandbox + production)

const BASE = 'https://open.tiktokapis.com/v2';

async function refreshIfNeeded(credentials) {
  if (Date.now() < credentials.token_expires_at - 300000) return credentials;
  const res = await fetch(`${BASE}/oauth/token/`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: new URLSearchParams({
      client_key: credentials.client_key, client_secret: credentials.client_secret,
      grant_type: 'refresh_token', refresh_token: credentials.refresh_token,
    }),
  });
  const d = await res.json();
  if (!d.access_token) throw new Error('Token refresh failed');
  return { ...credentials, access_token: d.access_token, refresh_token: d.refresh_token,
             token_expires_at: Date.now() + d.expires_in * 1000 };
}

async function queryCreatorInfo(token) {
  const res = await fetch(`${BASE}/post/publish/creator_info/query/`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}`, 'Content-Type': 'application/json; charset=UTF-8' },
  });
  const d = await res.json();
  if (d.error?.code !== 'ok') throw new Error('Creator info: ' + d.error?.message);
  return d.data;
}

async function initUpload(token, mediaUrl, caption, creatorInfo, isSandbox) {
  const endpoint = isSandbox
    ? `${BASE}/post/publish/inbox/video/init/`
    : `${BASE}/post/publish/video/init/`;

  const body = { source_info: { source: 'PULL_FROM_URL', video_url: mediaUrl } };

  if (!isSandbox) {
    body.post_info = {
      title:                    caption,
      privacy_level:            creatorInfo.privacy_level_options?.[0] || 'SELF_ONLY',
      disable_duet:             creatorInfo.duet_disabled    || false,
      disable_comment:          creatorInfo.comment_disabled || false,
      disable_stitch:           creatorInfo.stitch_disabled  || false,
      video_cover_timestamp_ms: 1000,
    };
  }

  const res = await fetch(endpoint, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${token}`, 'Content-Type': 'application/json; charset=UTF-8' },
    body: JSON.stringify(body),
  });
  const d = await res.json();
  if (d.error?.code !== 'ok') throw new Error('Init upload: ' + d.error?.message);
  return d.data.publish_id; // PULL_FROM_URL: no upload_url needed
}

async function pollStatus(token, publishId, isSandbox) {
  let delay = 3000;
  for (let i = 0; i < 20; i++) {
    await new Promise(r => setTimeout(r, delay));
    const res = await fetch(`${BASE}/post/publish/status/fetch/`, {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${token}`, 'Content-Type': 'application/json; charset=UTF-8' },
      body: JSON.stringify({ publish_id: publishId }),
    });
    const d = await res.json();
    const s = d.data?.status;
    if (s === 'PUBLISH_COMPLETE')
      return { success: true,  postId: d.data.publicaly_available_post_id?.[0] };
    if (s === 'SEND_TO_USER_INBOX' && isSandbox)
      return { success: true,  postId: null, note: 'Sandbox: draft sent to TikTok inbox' };
    if (s === 'FAILED')
      return { success: false, error: d.data?.fail_reason || 'failed' };
    delay = Math.min(delay * 1.5, 30000);
  }
  return { success: false, error: 'Status polling timed out' };
}

/**
 * Main export for PostHive
 * @param {object} credentials  - from connected_platforms.credentials (jsonb)
 * @param {string} mediaUrl     - public Supabase Storage URL of the video
 * @param {string} caption      - post caption / title
 * @returns {{ success: boolean, postId?: string, error?: string, note?: string }}
 */
export async function postToTikTok(credentials, mediaUrl, caption) {
  try {
    const creds       = await refreshIfNeeded(credentials);      // Step 0
    const creatorInfo = await queryCreatorInfo(creds.access_token); // Step 1
    const publishId   = await initUpload(                          // Step 2
      creds.access_token, mediaUrl, caption, creatorInfo, creds.is_sandbox
    );
    // Step 3 skipped — PULL_FROM_URL means TikTok fetches from Supabase URL
    return await pollStatus(creds.access_token, publishId, creds.is_sandbox); // Step 4
  } catch (err) {
    return { success: false, error: err.message };
  }
}
